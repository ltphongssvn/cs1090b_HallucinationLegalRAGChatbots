# Stack: Python/uv + PyTorch + Transformers + FAISS — OOD GPU Cluster (4x NVIDIA L4)

## Secret Management

### 1. Pre-commit Secret Detection
```bash
# Install hooks (run once after cloning)
uv run pre-commit install && uv run pre-commit install --hook-type pre-push
```

### 2. Environment Variables
`.env` is generated by `bash setup.sh` — **never commit it**.
Use `.env.example` as a template for collaborators:
```bash
export PYTHONHASHSEED=0
export CUBLAS_WORKSPACE_CONFIG=:4096:8
export TOKENIZERS_PARALLELISM=false
export RANDOM_SEED=0
```

## Security Measures

| Layer | Tool/Practice | Purpose |
|---|---|---|
| Secret Detection | detect-secrets + pre-commit | Prevent secrets entering codebase |
| .env Protection | `check-env-files` hook | Block .env commits |
| Credential Blocking | `block-cluster-credentials` hook | Block SSH keys, .pem, .netrc |
| Model File Blocking | `block-large-files` hook | Block .pth, .pkl, .h5, .safetensors |
| File Size Limit | `check-added-large-files` | 500KB max per file |
| Dependency Scanning | pip-audit in pre-push + CI | Scan for CVEs (non-blocking for pinned CUDA deps) |
| SBOM Generation | cyclonedx-bom in CI | Software Bill of Materials for compliance |
| Python Linting | ruff + mypy | Type safety and code quality |
| Test Coverage | pytest 80% per-file | Enforced via pre-push hook and CI |
| Reproducibility | uv.lock + frozen installs | Deterministic dependency resolution |

## Pre-commit Hooks (every `git commit`)

| Hook | Purpose |
|---|---|
| detect-secrets | Scan against `.secrets.baseline` |
| block-large-files | Reject model binaries (.pth, .pkl, .h5, .onnx, .safetensors) |
| check-env-files | Block .env commits |
| block-cluster-credentials | Block SSH keys and tokens |
| verify-selective-staging | Display staged files for review |
| mixed-line-ending | Enforce LF line endings |
| end-of-file-fixer | Ensure files end with newline |
| trailing-whitespace | Remove trailing whitespace |
| check-json / check-yaml | Validate syntax |
| check-added-large-files | 500KB max |
| check-merge-conflict | Block conflict markers |
| ruff-check + ruff-format | Lint and format Python |
| mypy | Type check src/ |
| pytest-quick | Run unit tests (fast gate) |

## Pre-push Hooks (every `git push`)

| Hook | Purpose |
|---|---|
| pytest-coverage-push | unit + contract tests with 80% coverage enforcement |
| security-audit-push | pip-audit CVE scan (non-blocking for pinned CUDA deps) |

## Coverage Enforcement (80% Per-File Minimum)
```toml
# pyproject.toml
[tool.coverage.run]

branch = true
[tool.coverage.report]
fail_under = 80
show_missing = true
```

## Known CVE Policy

`transformers<4.40` is pinned for CUDA 11.7 wheel compatibility with the L4 cluster driver.
Known CVEs in this range are accepted until the cluster upgrades to a CUDA 12.x-compatible driver.
Track: `logs/pip-audit.json` (generated each CI run and pre-push).

## Dependency Scanning
```bash
# Manual audit
uv run pip-audit

# Generate SBOM
uv run python -m cyclonedx_py environment --of JSON --output-file logs/sbom.json
```

## Team Guidelines

1. Run `uv run pre-commit install && uv run pre-commit install --hook-type pre-push` after every fresh clone
2. Never commit `.env`, SSH keys, or model weight files
3. Use HF Hub or DVC for model files >500KB
4. Review `.secrets.baseline` changes carefully before approving PRs
5. Rotate any accidentally exposed tokens immediately
6. Run `uv run pip-audit` regularly between CI runs
7. Keep `uv.lock` committed and regenerate deliberately: `uv lock && git add uv.lock && git commit`
